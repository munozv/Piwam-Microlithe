Piwam: Guide d'utilisation

Pré-requis

Cette documentation part du principe que vous possédez une installation fonctionnelle de Piwam. Si ce n'est pas le cas, veuillez vous référer à la documentation d'installation ou voyez avec la personne qui a mis en place l'application.
Dans la suite de cette documentation l'adresse http://piwam.serveur.org fera référence à la page d'accueil de votre Piwam.

Création d'une nouvelle association

Après une installation vierge, la page d'accueil vous propose de vous identifier ou d'enregistrer une nouvelle association. Comme vous ne pouvez pas encore vous identifier, cliquez sur "enregistrer une nouvelle association".

L'enregistrement d'une nouvelle association se passe en 2 étapes :

  1. Enregistrement des informations relatives à l'association
  2. Enregistrement des informations relatives à l'utilisateur qui est en train d'enregistrer la nouvelle association (vous).

Un minimum d'information est nécessaire : nom de l'association, votre nom et prénom, votre adresse e-mail, votre statut dans l'association, un nom d'utilisateur et un mot de passe. Les autres informations sont optionelles, et pouront être renseignées ultérieurement.

Identification

Vous êtes maintenant en mesure de vous identifier afin d'accéder aux différentes fonctionnalités de Piwam. Rentrez votre nom d'utilisateur et votre mot de passe indiqués précédemment.

Si vous n'arrivez pas à vous identifier, veuillez reporter le problème sur le site du projet.

Gestion des membres

Ajouter un nouveau membre

Il suffit de remplir les informations du formulaire. Les champs obligatoires sont le nom, le prénom, le statut au sein de l'association et la date d'inscription.

Un identifiant et un mot de passe sont requis pour le premier membre seulement (vous ?). Si le mot de passe est laissé vide, le membre ne pourra pas accéder à Piwam.

Modifier les informations d'un membre

Dans la rubrique gestion des membres, il y a une petie icône représentant un stylo : . Cliquez dessus. Vous obtiendrez alors le même formulaire que lors de l'ajout d'un nouveau membre. Vous pouvez modifier toutes les informations.

Restrictions

Tout le monde ne peut pas modifier le statut ou l'exemption de cotisation d'un membre. Des droits sont en effet requis :

  • Pour éditer un statut, il faut avoir le droit d'ajouter un statut
  • Pour éditer l'exemption, il faut avoir le droit d'enregistrer de nouvelles cotisations

Gestion des statuts

Par défaut, Piwam créé automatiquement les statuts classiques qu'on retrouve dans la plupart des asociations :

  • Président
  • Trésorier
  • Secrétaire
  • Membre d'honneur
  • Membre actif

Cette liste de statuts est utilisé lorsque vous souhaitez enregistrer un nouveau membre. Vous pouvez bien entendu en créer de nouveaux aisément, si vous souhaitez pas exemple un statut de vice président.

Les cotisations

Les types de cotisations

Lors de l'installation de Piwam, il y a une information qui reste manquante : le montant des cotisations. C'est ici que vous allez pouvoir les configurer. Piwam vous laisse la possibilité de gérer plusieurs types de cotisation. Plusieurs raisons à cela :

  • Dissocier les cotisations des personnes physiques de celles des personnes morales
  • Enregistrer une nouvelle configuration de cotisation chaque année, par exemple si le montant évolue.

Gestion des cotisations

Vous ne pouvez enregistrer une cotisation seulement si vous avez configuré au moins un compte et un type de cotisation.

Gestion des comptes

Lors de l'installation de Piwam, un "compte" est automatiquement ajouté, il s'agit d'une caisse de monnaie.
Vous pouvez bien entendu enregistre de nouveaux comptes, par exemple vos comptes bancaires, vos comptes Paypal ™, etc...

À quoi sert le champ "référence" ?

Le libellé complet du champ peut être très long, par exemple :

	> Compte UDF numéro 3278-3287-4398-001389987

La référence est une version courte et personnalisée du libellé, pour éviter d'afficher la version longue dans les champs de selection du compte pendant l'enregistrement d'une dépense, ou dans les bilans.
Pour le compte en exemple précédent, une référence possible pourrait être :

	> UDF_ASSOC

Ce qui facilitera ainsi grandement la lecture dans les autres écrans de l'application.

Gestion des activités

Qu'est-ce qu'une activité ?

Une activité peut représenter un évênement particulier, une activité récurrente ou tout autre chose vous permettant de segmenter la vie de votre association.

Les activités apparaîtront dans le bilan qui permettra ainsi de déterminer quelles activités ont rapporté de l'argent à l'association.

Gestion des dépenses et recettes

Les 2 rubriques sont parfaitement symétriques. L'une permet très logiquement d'enregistrer les dépenses effectuées par l'association, tandis que l'autre va permettre d'enregistrer les entrées d'argent.
